## Tarefa

Crie uma página para vincular produtos ou serviços a uma organização

## Requisitos técnicos

- Crie o formulário em @app\organization\link\page.tsx.
- Essa tela deve ser um server component
- Receberá uma organização como parâmentro.
- Terá labels para visualização dos dados da organização.
    Exemplo:
    Empresa/prestador: 2S Inovação
    CNPJ: 41704872/0001-18
    Endereço: Rua 12, nº 420, Noroeste.
    Telefone: (61) 98157-8965
    Contato: João da Silva
- Crie componente radiogroup (@components\ui\radio-group.tsx) para selecionar "produto" ou "serviço":
- Crie um campo de busca para a seleção do radiogroup.
- O campo de busca deverá ser code complete.
- Utilize a função `getProducts`, em @app\_lib\api\fetch-generated\index.ts, para buscar os dados dos produtos.
- Utilize a função `getServices`, em @app\_lib\api\fetch-generated\index.ts, para buscar os dados dos serviços.
- Crie os campos de cadastro de um produto.
- Crie os campos de cadastro de um serviço.
- Crie um compoente Link do next para renderizar os campos de cadastro de um produto ou serviço.
- Utilize o componente Card em @components\ui\card.tsx para agrupar os campos do formulário.
- Crie um cabeçalho na página do formulário utilizando o componente `header` em @components\header.tsx.
- Para vincular um produto a uma organização, utilize a função `createOrganizationProduct` em @app\_lib\api\fetch-generated\index.ts.
- Para vincular um serviço a uma organização, utilize a função `createOrganizationService` em @app\_lib\api\fetch-generated\index.ts.
- Use o zod para validação dos campos do formulário.
- Utlize os estilos css em @app\globals.css.

## Regras de negócio

- Ao selecionar o componente radiogroup (produto ou serviço), deverá ser renderizado o campo de busca.
- Para renderizar os campos de cadastro de produto ou serviço, deverá ser utilizado o link criado e será renderizado somente os campos específicos para cadastro do componente selecionado.
- Ao selecionar o resultado do campos de busca, o id do produto ou serviço selecionado deverá ser armazenado para posterior envio como parâmetro na URL.
- Caso não seja selecionado nenhum produto os serviço, o produto e serviço será cadastrado e posteriormente vinculado a organização.
- Após o retorno positivo, redirecionar para a página /assessment passando o `id` da organização como parâmetro.
